## 4. Vue 生命周期 & 浏览器关闭后

**Vue2 生命周期**：
1️⃣ 创建阶段
beforeCreate
data、methods 都不能用
很少使用
created
data、methods 已初始化
不能操作 DOM
常用：初始化数据、发请求（不依赖 DOM）
2️⃣ 挂载阶段
beforeMount
模板已编译
DOM 还未挂载
mounted ⭐⭐⭐
DOM 已渲染完成
常用：DOM 操作、第三方库、获取元素
3️⃣ 更新阶段
beforeUpdate
数据已变
DOM 未更新
updated
DOM 已更新
避免在这里改数据（容易死循环）
4️⃣ 销毁阶段
beforeDestroy
组件即将销毁
常用：清定时器、解绑事件
destroyed
组件已销毁
所有绑定已清除
beforeCreate → created → beforeMount → mounted → beforeUpdate → updated → beforeUnmount → unmounted
Vue 生命周期分为创建、挂载、更新、销毁四个阶段。
Vue2 常用 created 初始化数据，mounted 操作 DOM，beforeDestroy 做清理。
Vue3 使用 setup 替代 created，并通过 onMounted、onUnmounted 等钩子管理生命周期。

**浏览器关闭后**：

* Vue 实例销毁
* JS 线程终止
* 内存释放
* WebSocket / 定时器全部断开

## 6. WebSocket 是否跨域，如何解决

* **WebSocket 本身支持跨域**
* 不受浏览器同源策略限制
* 服务端校验 `Origin` 即可
* 不需要 CORS

当网页尝试请求 不同源（域名、协议、端口任意一个不同） 的资源时，浏览器出于安全策略，会阻止请求，这就是跨域问题。

CORS（服务器端设置） — 最推荐
服务端响应头中加入：
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, PUT, DELETE
Access-Control-Allow-Headers: Content-Type, Authorization
支持 预检请求（OPTIONS）
安全可控，面试常问
