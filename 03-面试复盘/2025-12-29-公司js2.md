每个 HTML 元素都是一个盒子，由 四部分 组成：

+---------------------------+
|        Margin             | 外边距
+---------------------------+
|        Border             | 边框
+---------------------------+
|        Padding            | 内边距
+---------------------------+
|        Content            | 内容区
+---------------------------+


盒模型 = Content + Padding + Border + Margin
content-box → width 只算内容
border-box → width 包含 padding + border


CORS 是浏览器跨域安全机制，后端通过响应头控制允许的跨域源、方法、请求头和是否携带 cookie。


预检请求（Preflight Request）
浏览器在发送实际请求前，先发 OPTIONS 请求，询问服务器是否允许该跨域操作（方法/头部）


强缓存：浏览器直接使用，不发请求 → Cache-Control, Expires
协商缓存：浏览器先发请求，服务器返回 304 → Last-Modified/ETag
优先级：强缓存优先，其次协商缓存


箭头函数语法简洁，不绑定自己的 this

URL 到浏览器渲染的完整流程(简短版)
浏览器渲染流程：URL → DNS → TCP/HTTPS → HTTP 请求 → 构建 DOM + CSSOM → 渲染树 → 布局 + 绘制 → JS 执行


v-if 控制 DOM 是否渲染 → 高开销，适合少切换；
v-show 控制 display 样式 → 低开销，适合频繁切换。


改变几何信息 → Reflow → Repaint → Composite
改变外观 → Repaint → Composite
动画/动效用 transform/opacity → 只 Composite（性能最优）


prototype(普若特泰普)原型链是 JavaScript 实现继承的核心机制
