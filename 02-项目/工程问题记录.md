## 工程实践与问题解决



### GitHub 推送失败问题排查（HTTPS → SSH）

**问题现象：**
- 在 Windows + PowerShell 环境下使用 `git push` 推送至 GitHub
- 多次出现 `Recv failure: Connection was reset`
- 包括使用 GitHub 官方 HTTPS 及 ghproxy 镜像均无法成功

**排查与解决过程：**
1. 排除 Git 基础配置问题（user.name / user.email）
2. 判断为网络环境对 GitHub HTTPS 流量进行重置
3. 切换方案：使用 SSH Key 进行身份认证
4. 生成并配置 ed25519 SSH Key
5. 将仓库 remote 地址由 HTTPS 修改为 SSH
6. 成功完成推送.

**收获：**
- 理解了 HTTPS 与 SSH 在 Git 认证与网络层面的差异
- 掌握了在受限网络环境下的 GitHub 访问解决方案
- 提升了对真实工程环境问题的定位与解决能力
